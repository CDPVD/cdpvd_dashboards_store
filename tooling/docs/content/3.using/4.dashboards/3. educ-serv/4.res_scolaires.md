::alert{type=info}
Ce tableau de bord fournit un aperçu des résultats des écoles de votre CSS à différentes matières et compétences, à chaque étape et au bilan de l'année scolaire choisie. Il montre aussi les écarts de résultats de l'ensemble des écoles par rapport au CSS, par étape et au bilan. 
::

# Déploiement
:badge[tag:res_scolaires]{type="success"}
:badge[new in v0.7.0]

## Base de données

La base de données `gpi` doit être liée au projet. Référez-vous à la section [linking a database](/using/configuration/linking) pour davantage d'information sur cette procédure.

## Comptoir 

Le comptoir suivant doit être activé pour que ce tableau de bord soit fonctionnel. Référez-vous à la section [enabling a mart](/using/configuration/enabling) pour davantage d'information sur cette procédure.
* `educ_serv`

## Spécificités de ce projet Dbt 
> Mettez à jour le fichier `cssXX.dashboards_store/dbt_project.yml` en utilisant l'extrait suivant:

```yaml
models:
  core_dashboards_store: # Enable the models from the core repo
    marts:
      educ_serv:
        +enabled: True                  
    dashboards:                                   
      res_scolaires:
        +enabled: True
    interfaces: # The dashboard only needs the GPI database
      gpi:
        +enabled: True
```  

# Configuration
::alert{type=warning}
La configuration est optionnelle. Si vous ne faites pas de configuration, le tableau de bord utilisera celle par défaut.
::

## Personnalisation des matières suivies
> Mettez à jour votre fichier `cssXX.dashboards_store/dbt_project.yml`.
> Cette table nécessite l'utilisation de fichiers de données statiques (seeds). Assurez-vous de rouler `dbt seed --full-refresh` pour les alimenter.

* Pour ajouter une liste de matières à suivre:
  1. Ajoutez un fichier `.csv` dans votre dossier `cssXX.dashboards_store/seeds/dashboards/res_scolaires`. Ce fichier doit se nommer `custom_matiere.csv`. Il doit être alimenté grâce aux colonnes décrites par `core.dashboards_store/seeds/dashboards/res_scolaires/schema.yml` (fait référence à la seed `custom_matiere`). 

  2. Déclenchez un rafraîchissement de vos fichiers de données statiques. 

```bash
dbt seed --full-refresh
```

## Définir un seuil personnalisé à atteindre: `threshold`
> Le seuil (threshold) est utilisé pour comptabiliser le pourcentage d'élèves l'atteignant ou le dépassant. Il est programmé à 70% par défaut.

Vous pouvez surcharger ce seuil par défaut en ajoutant la variable suivante à votre fichier `dbt_project.yml`.

```yaml
# cssXX.dashboards_store/dbt_project.yml
vars:
    # res_scolaires's dashboard variables:
    res_scolaires:
        threshold: 70
```

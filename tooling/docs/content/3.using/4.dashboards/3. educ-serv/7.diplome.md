::alert{type=info}
Ce tableau de bord suit les élèves de la 5e secondaire ayant la possibilité d'obtenir un diplôme d'études secondaires J5 à l'année scolaire en cours en tenant compte de toutes les exigences de la sanction des études.
::

::alert{type=danger}
Pour ce tableau de bord, les modèles de Sciance doivent être utilisés.
::


# Déploiement 
:badge[tag:diplome]{type="success"}
:badge[new in v0.11.0]

## Databases 

Les bases de données `GPI` et `Jade` doivent être liées au projet. Référez-vous à la section [linking a database](/using/configuration/linking) pour davantage d'information sur la procédure à suivre.

## Fichier de données statiques (Seed)

::alert{type=warning}
Vous devez faire une `seed` pour que ce tableau de bord fonctionne dans votre CSS.
::

Le tableau de bord requière un fichier de données statiques `matiere_evalue` pour être utilisé. Référez-vous à la section [seeding](/using/configuration/adapts-seeds) pour plus d'information permettant d'ajouter un fichier de données statiques à un comptoir (how to seed a mart). Ce fichier de données statiques est requis pour alimenter les matières à sanction. Comme vous n'aurez pas à le surcharger,vous devrez simplement rouler cette commande pour le créer: dbt seed --select +tag:diplome
Le fichier créé est décrit dans `core.dashboards_store/seeds/dashboards/diplome/schema.yml`.

## Comptoir

Le comptoir suivant doit être activé pour que le tableau de bord fonctionne. Référez-vous à la section [enabling a mart](/using/configuration/enabling) pour plus d'information sur la procédure à suivre.
* `educ_serv`

## Surchargement

Le tableau de bord utilise plusieurs filtres qui ne correspondront pas nécessairement à vos besoins. Si tel est le cas, vous devrez surcharger le dossier entier. (PEVR folder???) Référez-vous à la section sur le surchargement au besoin (overriding).

## Spécificité du projet Dbt
> Mettez à jour votre fichier `cssxx.dashboards_store/dbt_project.yml` en utilisant l'extrait suivant:
```yaml
# cssxx.dashboards_store/dbt_project.yml
models:
    store:
        dashboards:
            diplome:
                +enabled: True
        interfaces:
            gpi:
                +enabled: True
            jade:
                +enabled: True

vars:
  database_gpi: "The name / IP of the gpi database"
  database_jade: "The name / IP of the jade database"
```

::alert{type=info}
> Ce tableau de bord fournit un portrait global de l'année scolaire en cours et de l'année scolaire précédente d'une école.
> Il inclut un portrait annuel des élèves de l'école. Pour obtenir les résultats dans les compétences, vous devez activer la table suivante:`core.dashboards_store/models/dashboards/res_scolaires/pbi_tables/resco_report_res_bilan_comp_eco.sql`
> Enfin, il fait le suivi des résultats des élèves dans les matières sélectionnées. Il illustre l'historique des résultats des élèves présentement inscrits au primaire ou au secondaire dans les matières sélectionnées, sur une période allant jusqu'à 5 ans. Le tableau peut produire une liste des élèves selon qu'ils soient en échec, en difficulté ou en maîtrise dans les matières de base, à l'année scolaire en cours ou à l'année scolaire précédente.
::

# Déploiement
:badge[tag:suivi_resultats]{type="success"}
:badge[new in v0.10.0]

## Base de données 

Le base de données `GPI` doit être liée au projet. Référez-vous à la section [linking a database](/using/configuration/linking) pour plus d'information à ce propos.

## Tableaux de bord

Ce tableau de bord dépend du tableau de bord `res_scolaires`. Il faut donc avoir déployé le tableau de bord `res_scolaires` avant de déployer le tableau de bord `suivi_resultats`.

## Spécification du projet DBT 
> Mettez à jour votre fichier `cssXX.dashboards_store/dbt_project.yml` avec les extraits suivants et ajoutez la variable `cod_css`.


```yaml
#cssXX.dashboards_store/dbt_project.yml
models: 
  core_dashboards_store:
    marts:
        educ_serv:
            +enabled: True        
    dashboards: 
        educ_serv:

      		suivi_resultats:
        		+enabled: True
      		res_scolaires:
        		+enabled: True        
    interfaces:
      gpi:
        +enabled: True
```

# Configuration
::alert{type=warning}
La configuration est optionnelle. Si vous ne faites pas de configuration, le tableau de bord utilisera celle par défaut.
::

## Personnalisation des parcours suivis
> Cette table nécessite certains fichiers de données statiques (seeds). Assurez-vous de rouler `dbt seed --full-refresh` pour alimenter ces fichiers de données statiques.

* Pour ajouter une liste de cours internes à suivre:
  1. Ajoutez un fichier `.csv` dans votre dossier `cssXX.dashboards_store/seeds/dashboards/educ_serv/suivi_resultats`. Ce fichier doit se nommer `custom_tracked_courses.csv`. Le fichier doit être alimenté avec les colonnes décrites dans `core.dashboards_store/seeds/dashboards/educ_serv/suivi_resultats/schema.yml` (réfère à la seed `custom_tracked_courses`). Vous pourriez utiliser la table  `core.dashboards_store/models/marts/educ_serv/staging/stg_descr_mat` pour trouver la cartographie appropriée.
  2. Déclenchez un rafraîchissement de vos fichiers de données statiques. 

```bash
dbt seed --full-refresh
```

::alert{type=info}
Référez-vous à la section [seeds](/using/marts/seeds) pour plus d'information à propos de l'alimentation et de l'utilisation des fichiers de données statiques (seeds).
::

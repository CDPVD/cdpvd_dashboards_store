## Base de données

Les bases de données `gpi` et `jade` doivent être liées à ce projet. Référez-vous à la section [linking a database](/using/configuration/linking) pour davantage d'information sur la procédure à suivre.

## Comptoir 

Le comptoir suivant doit être activé pour que le tableau de bord soit fonctionnel. Référez-vous à la section [enabling a mart](/using/configuration/enabling) pour davantage d'information sur la procédure à suivre.
* `educ_serv`


#### Spécificité du projet Dbt
> Mettez à jour votre fichier `cssxx_store/dbt_project.yml` en utilisant l'extrait suivant:

```yaml
# cssXX.data.store/dbt_project.yml
models: 
  core_dashboards_store:
    marts:
      educ_serv:
        +enabled: True 
    dashboards:
      other:
        pevr:
          +enabled: true
    interfaces:
      gpi:
        +enabled: true
      jade:
        +enabled: true  
```
# Configuration
::alert{type=warning}
La configuration est optionnelle à l'exécution du code. Si vous ne faites pas de configuration spécifique, le tableau de bord utilisera la configuration par défaut.

## Personnalisation des indicateurs PEVR
> Cette table fonctionne avec des fichiers de données statiques (seeds). Assurez-vous de rouler `dbt seed --full-refresh` pour les alimenter.

* Pour ajouter une liste d'indicateurs locaux à suivre:
  1. Ajoutez un fichier `.csv` dans votre dossier `cssXX.dashboards_store/seeds/dashboards/pevr`. Le fichier doit se nommer `custom_indicateurs_pevr.csv`. Il doit être alimenté avec les colonnes décrites dans `core.dashboards_store/seeds/dashboards/pevr/schema.yml` (réfère à la seed`custom_indicateurs_pevr`). 
  2. Ajoutez un fichier`.csv` dans votre dossier `cssXX.dashboards_store/seeds/dashboards/pevr`. Ce fichier doit se nommer `custom_indicateurs_pevr_matiere.csv`. Il doit être alimenté avec les colonnes décrites dans `core.dashboards_store/seeds/dashboards/pevr/schema.yml` (réfère à la seed `custom_indicateurs_pevr_matiere`).   
  3. Déclenchez un rafraîchissement de vos fichiers de données statiques: 

```bash
dbt seed --full-refresh
```

::alert{type=info}
Référez-vous à la section [seeds](/using/marts/seeds) pour davantage d'information sur la procédure permettant d'utiliser et d'alimenter les fichiers de données statiques.
::
